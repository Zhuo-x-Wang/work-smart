标题：精明工作

# 立意

依托他方智慧，反思工作经验，制定提升工作效率和成效的技术工具箱。
 
## 立意的逻辑基

囿于人格特征，我的思与行在工作效率上体现一定缺陷。高效能工作上的素养养成，我亦级别尚浅。

他方智慧将从三个角度帮助我：
1）知识体系、技术结构；
2）实用的方法与技巧；
3）人格特征分布洞察及建议。
 
它还将促进我反思自己的工作经验，包括下列三大部分：
1）职业生涯；
2）学习；
3）生活。

一份指导、实用、可持续发展的技术工具箱，将发挥三点作用：
1）作为上述改进努力的MEMO（总结、归档、查阅）；
2）为我长期、持续、计划的提升起到支撑；
3）启发和帮助他人。

# 正文

## 引子

也许不只是汉语，一门语言正在堕化。我经常听说，人们赞扬对方“工作‘刻苦’”。它是在书面环境中快被滥用的褒义词。刻，指程度深。苦，有疼痛、有困住、有不开心，还可以延伸更多。现在，忘掉你认识这个词，花十秒钟，从零想象这两个字组成的东西——刻·苦。你得到了什么？
无非一种心理上的限制。别无其他。你的情感，当机；你的智能，当机；你能做什么，我打赌刚才你的脑袋压根没想过。我继续看到一个泄气的人，开始怀疑，他要解决什么问题，才害得他落入这番境地，可一无所知。

我没有发明“精明”这个词。但我确实从零发现某两个字的组合物——精·明。起因于我把“Work Smarter, not harder”译为“更精明地工作，而非刻苦”。我意图的逻辑是：精，代表机灵、敏捷；明，代表明白、智慧，总之即“更为巧干，而非蛮干”之意。
你和我可能耳濡目染，因人们总喜欢以暧昧的语气，用它形容一种了不起的商人或滑头的人，而对它持有偏向贬义之感。但在这里，我们且将摆脱世俗的干扰。
事实上，辞典释义：精细明察；精通明白；耳聪目明，精力旺盛；光明晴朗；纯洁至实。我认为，这些积极开放的表意，非常有利于强化我们以精明为挈子从而实现终极目的的工具箱。我会适当引入它们，丰富我最初的表意。

## 计划

### Q：它要干嘛？ 

计划的价值，仅提炼如下：
1）将理清问题和思路。这在真正做事（尤其复杂之事）之前，必不可少。
2）将树立过程目标，甚至依赖路径，这使成功通路变得清晰。
3）将用于过程检查，保证了行动、执行的持续、有效、高效。
4）作为程序化经验，帮助主体在未来类似或不同项目中取得更高的成功机率。 

Margaret Meloni：
- 每周，每天，最好都有个计划：想完成什么，需完成什么。不是说连每分钟也要有计划，但如果你要完成什么特别的事，计划能够帮到你。

### Q：怎么实现它？

我的重心，正是放在一份计划表的制订。我认为，不同场景，对计划的定义（即计划表的构成要素）的要求不一。需要权衡的主要在于：
1. 首要是时间。
举一个可能不恰当的例子，为完成此文（第一版），我愿意投入几乎无限的时间，因为我目前处于休假期，时间不是一项有效约束——从长远看当然仍然有限制，譬如我不可能花一个月甚至半年才完成它。
2. 能力很重要。
沿用上面例子，我自信此文第一版的完成，是付出工时便能水到渠成的事，同时，我需要给即兴创造保留足够空间，所以我弱化了排期的要求。
我会额外强调，时间和能力几乎是决定成事的所有因素的终极集合。能力是量与质的结合，能力不足可能直接导致时间的无穷大，即目标的无可实现。不是一切都能拿时间来衡量，有的地方它会失效。

进一步，参考如下几个考虑方面：
1）时间；
2）为取得什么，而需要做什么；
3）为取得什么及需要做什么，而需要什么资源； 
4）其他必要备注，比如察觉到的风险、推测的捷径、圈出利益相关者等。 

Margaret Meloni：
- 一份计划将体现：做什么，按怎样顺序做，要何时做完，如果遇到特殊情况该怎么办。
- 注意包含哪些要素，如优先级、最后期限、要采取的前提步骤、交付给谁的、花多久、需要什么信息、需要什么资源、打算何时开始等
- 制作模板，测试哪些要素好用，哪些不必要

### 延伸

表格是一门通用、基础的媒介、工具，它还将在项目管理（项目表、干系人登记表）等技术域中使用到。虽然此文不会特别花笔墨于解释底层工具的性质。

### 结语

我认为上述要点和样板，足够启发彼一时的自己和他人，去定义（包括试验改善）结合实际情况的计划要素，从思考过程和产出文件两方面。当然，有人可能会指出，不妨概括：一切有助于计划成功的考虑面，均可纳入计划表的单元列，对计划项目进行周密管理。我的评价是：思想虽好，实践起来却过分麻烦和降低ROI了。坚持实用的原则即可。所以我不再啰嗦计划表的构成要素。

类似地，精明工作的技术从一开始就未限定于以单个人为边界，基于技术的传播特性和协作作用，我认为不必要形式化地啰嗦说明任一技术对团队和组织的作用，例如：一份良好的计划，有助于多人协同的工作。除非跨个体应用确实对个人操作提出严肃的挑战，那么我会直接拎出来说。“计划”这一部分专注在个人领域，不牵涉“组织结构调整计划”“季度财务计划”

实际上，上表存在几个难点，包括如何定义优先级、如何评估所需时间等，将在后文专门展开。

了解计划的意义及重要性，在需要的时候，能够想起来制定计划，且经持续改善从而定义出恰当的计划要素，在实践当中共享和传播这一工具的价值，这是这一部分的核心价值，无论读者多少，对我实是如此。

## 优先级

### Q：为何它是个难题？

没有对错而言，我先摆上几条观点：
1. 不管要做的事，是否自己发起、控制，只是因为事情与事情交织在一起，就不得不搞清楚各自的优先级。
2. 越完全属于自己的事，我越有自主决定权，更重要的是，我更容易清楚和应用评价的标准，从而选择机动、拖延等。当然只要很多事情交织到一起，局面仍会失控。
3. 现实中遇到更多的，是那些来自他人的指配的事（这在职场和社交上常发），他们可能希望我自己去定，或他们根本不知道事情之间优先级怎样，更不晓得对于我，应该采取哪种安排才算科学和正确。 

即使度量方式为主观评估，却仍然能够应用科学方法。我暂且提取出如下原始变量，它们会增加或减轻一份整体的优先级评级工作的困难程度：
1）事情的数量（越多越难，下面一概以“正”简称）；
2）事情的类型（正）；
3）事情对应的利益相关者（正）；
4）我对事情的了解（负）；
5）领导的决定性意见（负）；
6）组织的价值观或制度（负）；
7）我的情绪状态（负）；
而影响单个项目的优先级进而可能与其他项目竞争胜利的经典变量：
1）紧急程度（越高越优先，下面一概以“正”简称）；
2）重要程度（正）。

在我认为，优先级是个判定问题。这意味需要考虑的因素，或衡量标准的多样性或复杂度，极大影响到解决的难易。

### Q：如何处理它？

解决判定问题的思路有不少，概括起来包括：
1. 对事实更多的了解；
2. 更简单的原则；
3. 量化；
以及辅助但十分必要的技术（体现为文化、工具、制度等）。

Margaret Meloni提供的直接建议有：
- 聆听来自领导阶层的声音
- 聆听来自顾客的声音，支持顾客
- 仔细倾听周围的声音，留意会议、通告或其他沟通过程中什么讨论最多，留意取得成功的同事们把时间花在哪，它们是你待发现的线索
它们具体而实用。但我补充：

举一个真实的例子，我的收件箱有时会收到客服部门要求3小时内回复的客诉邮件，异常检查有时需要产品人员和开发人员且可能跨产品线（这意味跨子部门或团队）。该响应政策，以红字惯例出现在每个案例的首次报告邮件的内容顶部。但我从来没有见到该政策其他形式的传播工作。直白说，红字提醒了读者，但读者未必把它看作他们需要贯彻的政策。虽然绝大多数案例在人工作业阶段的耗时很低，但在此之前也即之后，存在不可预料的等待作业期——3小时的大时间窗口中，传递节点越多往往带来越多响应空白时段，比如我开1小时的会而未处理任何邮件，此案等于报告后1小时内一直处于无人受理状态。等我回到邮箱，我不得不迅速、优先地处理它。尴尬的是，有的案例我评估不出是紧急或高影响，有的甚至没有说明是否对顾客造成影响，于是在红字警戒与允许余地的双重作用下，我一直只能以模糊的响应策略来应对该类工作。我的下个节点，又任何呢？因此有时我不无担忧，甚至有时进行了自认为逾越职责或不必要乃至是否有些神经质的督查行为。
我描述的是运营领域一类很典型的问题。很难想像，光靠单个员工应对优先级的技艺，而不改善整体策略，就能够缓解诸人的优先级困境。

因此，这里我极度强调，问题域越偏向公众领域，越需要“群策”的过程。应该注意到，Meloni的三条建议均非“独狼”型。

抱着实用、具体、可操作的定位，我在此补充列出更多对策：
1）思考、提问我们的事业是什么，从组织成功经验、管理层、组织战略、自身经验等寻找衡量标准；
2）做好功课，对待排优先级的事情，要有足够的了解，熟悉它的工作，甚至提前预估风险等；
3）更多人际互动，与上级、同事、团队成员、客户等对话，听听他们的意见或诉求，也有利于达成1）2）；
4）应用和贡献组织的知识基、汲取和影响组织的决策智慧，以寻找或形成更加系统成熟或更为简单可靠或方便量化辅助的判定策略。

### 延伸

优先级是非常通用的管理工具。例如，在交互设计中，把待改善的点进行条目化登记后，有必要标注优先级，从而决定开发、解决的次序。我会在其他问题域的技术工具箱中，给出包含优先级及评估说明的管理登记表。特定域展现独特的优先级原则，那是非常的精彩。

### 结语

我意识到有趣的一点：优先级的结果，是个量化值，无论高中低或12345。该量化意义重大。而取得量化的方法，又分主主观评估或定量计算。有时，过于复杂的、结合成本、收益等要素及权重计算出来的优先级反倒不必要。如果个别要素特别重要，那么在条目中值得单独列出，而优先级很可能是复杂现实和定性因素下，结合人性因素所估定出来的、利落的结果。尤其在通用非专业领域，提升工作成效和效率上，我不推荐使用复杂的方法。当拥有优先级概念，能够使用它提高管理和行动的秩序，便已是了不得的素养了。

### 权衡调控

按照真实生活，唯一一张时间日志表，将记录包括家庭生活、职业工作、社交等诸多活动。我的注意力决定了该表对我的价值。有人会在意工作和家庭间的平衡，有人可能着重希望分析工作领域，有人在意一定要给社交保留足够的时间。所以，第一点，你关心的是什么？个人的理想标准是什么？

第二点，个人的实际分布是什么？找到目标与预期间的差距在哪（什么类别及什么原因导致）？Margaret Meloni给的工具是：策略预留时间（Strategic Reserve Time）。

我的看法，必须批判性看待SRT正、负、零：
1. Strategic Reserve Time＝Time available to do all tasks - all commitments，它代表剩余多少时间可以用来做新的或不同的事情。
通过持续改进工作效率，使减数变小，所取得的差额即SRT>0，是好事。
1）投入计划外事务，如充个电，如提前浏览本安排在明天的报告；
2）应急意料外的紧急工作，如领导突然安排个活；

2. Strategic Reserve Time＝Time expected based on basic job function - all commitments，它代表预期与实际的不符。
由于错误估时或不合理分配，使被减数虚高或虚低，所造成的余额或负债即SRT≠0，显露计划管理的困境（也正是SRT测算意义）。

## 沟通管理 

沟通是极其广泛的话题，考虑邮件为最普遍的沟通工具，且缩小到个人生产力范围，目前仅谈邮件：

### 邮件

Margaret Meloni：
1）定时查收邮件，而不是随时；
2）无需你干涉的，考虑删除或存档；
3）分类文件夹；
3.1）情报类邮件，随手存档；
3.2）行动类邮件，在工作规划和日程表中存档、排期；
3.3）回复类邮件，存档至待办，设置日程提醒器；

自我简评：
1. 很遗憾，似乎没有简单的方法统计我对邮件的响应表现如何，如邮箱接收、我查阅、我回复三者的时差，如查阅或回复行为的时间点趋于集中还是过多分散。印象中，我的响应速度有时过快，频率较高。这应当改善。
2. 在文件管理上，表现要好一些，我有几条不错的习惯：
1）利于Outlook规则、邮件文件夹，按来源自动归类邮件；
2）使用颜色标记特殊邮件；
3）不定时存档某一重要主题的重要邮件。

在结合自身经验和Meloni的提示下，我创建了如下综合策略：

1. 在线管理
1.1 按规则自动将特定来源的邮件接收至各文件夹
    ——按角色或部门，自动；讲究快速辨识和机器初筛；software folder，目标文件夹稳定
1.2 定期将特定性质的邮件.msg文件快速分类至情报/行动/回复/丢弃等4类文件夹
    ——按性质，手动，拖拽即可；讲究快速管理，很可能暂不细看；software folder，目标文件夹稳定；可标识紧急度或截止时间，后续可标识是否完成
1.3 定期将特定主题的邮件.msg文件辨识归类至各文件夹
    ——按内容，手动；讲究可持续管理，易备份至线下；software folder，目标文件夹需恰当设置，不要太深，会不定时增减；可运用颜色标识）；

2. 离线管理
2.1 利用1.2和1.3将重要邮件.msg文件不定时存档
    ——何谓重要无固定标准，手动，复制粘贴；讲究易提取和备份；system folder，目标文件夹稳定且有新的扩增；由于脱离software因此无任何标识；建议线下线上文件夹映射关系为：高级尽量一致但尊重线上、反思线下，低级以线下为准、线上不必仿佛但可借鉴、反思线下；不建议随手删除线上文件
2.2 按日期将收件箱邮件备份为.pst
    ——包括1.1、1.2、1.3，不包括已删除邮件，不会遗漏2.1

以上虽然尚未在职业工作场合中检验，但对我，预计实施起来应当自然而然，因既扎实建立在过去1~2年的经验之上，且吸收了新的思想技巧转化为可操作。我很乐意在未来继续跟踪和维护上述策略，让实践来证明吧。
 
## 多任务处理

本部分特别感谢由Meloni引荐的Mike Sanders，主要知识来自他在Youtube发布的《Advanced Multitasking》系列视频。

我承认即使从事总结和学习的近期，我掉入过多任务处理的陷阱。

### Q：多任务处理伴随什么难题？

我相信许多人和我一样，听说过“Flow（心流）”。事实上，我对它不怎么了解。我只是主观感受过它所指的沉浸于某一活动的心理感觉，似乎有种幸福感，充实，头脑机灵，一切变得简单和从流。我想说的是，它的反面或敌人：从一个活动切换到另一活动的Switch cost（切换成本）。（注：这并非心理学上的公认，我只是把它俩联系起来，形成对立。）
比如编制产品设计文档时，突然来了电话，项目经理通知我需要临时参与某个会议，帮助解答某个疑问。即使只是1刻钟的来回，但我的设计思考和文书创作被打断，再次回到座位上我可能忘记进入到什么位置，或丢失了一些点，继续的效率大不如前。
容易想到，降低切换成本的方法之一：对切换前的任务进行标记，使得你切换回该任务时迅速进入状态，包括：
1）备忘当前进度；
2）草稿化地记下当前所想到的下一步的要点等。

这是个技巧性方法。

### Q：真的需要多任务吗？
Mike Sanders的眼光从个人生产力突破进入管理学的范畴。他的建设性意见：
1. 删掉你不该做的
2. 委托、授权（但需要训练他人）
3. 转移、传递（但需要礼貌）
4. 修剪范围、缩减范围

多任务处理是一个管理性质的活动，多任务处理是一定观察尺度下的现象，因为人做不到同时做几件事。而切换成本是必然存在的，一定尺度下的多任务处理表现也是有弊端的。实际上，在时间管理等领域，管理学研究者纷纷指出累积大段时间集中处理一件事的重要意义，不过这里不作展开。

Mike Sanders对Multi-tasking的定义是：Switching complex tasks frequently in an attempt to do more than one thing at the same time.
他提出的Advanced Multi-tasking则是：Performing validated. prioritized tasks exquisitely, one task at a time.
很有意思，待会从下面的他本人绘制的策略图解可进一步理解。

“Do more, work less”是一简单到位的概括。

这些是从源头入手的方法。

在他的提醒下，我迅速对近期的诸多项目进行罗列和分类，正式和利落地给自己一个交待，哪些是优先级极高的，哪些是低的，哪些是已经证实可以放弃或已经放弃的。这极大缓解了我之后的工作效率，专一带来了工作成果的迅速累积。

### 结语

多任务处理与计划、优先级联系紧密，但总的来说，计划和优先级倾向于规划和参考，而多任务处理才是真正的执行范畴的话题。因此，容易理解，即使制订完善的计划，也不能保证人在实际作业中又陷入失秩的麻烦。高效的多任务处理本领，让实施变成对计划的高效执行。必须掌握它。
